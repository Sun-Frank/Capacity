# CAPICS 端到端自测清单（生产部署版）

## 文档信息
- 更新日期：2026-04-25
- 文档版本：v1.2.2

## 0. 测试前提

- 已执行 `deploy/deploy.sh` 且无报错。
- `.env.prod` 中已配置可用的数据库账号、JWT 密钥、域名与 CORS 白名单。
- 首次初始化后默认管理员账号为 `admin / admin123`（如 `schema.sql` 保持默认）。

## 1. 一键部署执行检查

1. 复制参数模板并修改：
   - `cp deploy/.env.prod.example deploy/.env.prod`
2. 执行部署：
   - `bash deploy/deploy.sh deploy/.env.prod`
3. 检查关键输出必须包含：
   - `Deploy completed.`
4. 检查后端服务状态：
   - `systemctl status capics-backend --no-pager`
   - 预期：`active (running)`

## 2. 后端连通性检查

1. 直连后端健康检查：
   - `curl -fsS http://127.0.0.1:8080/api/health`
   - 预期：JSON 含 `"status":"UP"`
2. 通过 Nginx 反代检查：
   - `curl -fsS http://<你的域名或IP>/api/health`
   - 预期：同样返回 `"status":"UP"`

## 3. 认证链路检查

1. 登录：
   - `POST /api/auth/login`
   - Body:
   ```json
   {
     "username": "admin",
     "password": "admin123"
   }
   ```
2. 预期：
   - HTTP 200
   - 返回体包含 `token`
3. 用 token 调用：
   - `GET /api/auth/current`
   - Header: `Authorization: Bearer <token>`
4. 预期：
   - HTTP 200
   - 返回用户信息

## 4. 前端发布检查

1. 打开首页：
   - `http://<你的域名或IP>/`
2. 预期：
   - 页面可加载，无白屏
   - 浏览器 Network 中静态资源返回 200
3. 登录后进入核心页面（建议至少验证以下）：
   - 仪表盘 `/`
   - 静态产能评估 `/capacity-assessment`
   - 动态产能模拟 `/capacity-realtime`
4. 预期：
   - 页面接口返回 200 或业务可解释错误
   - 无持续性 401（过期 token 除外）

## 5. 数据与功能冒烟检查

1. 导入一份最小可用主数据（编码族/产品/MRP 任意一组测试数据）。
2. 打开仪表盘或产能评估页面，确认有业务数据展示。
3. 在动态模拟页调整一个参数并保存快照，再加载快照确认回显。

## 6. 自动化自测命令（推荐）

执行：

```bash
bash deploy/e2e-smoke.sh deploy/.env.prod
```

预期：

- 输出 5 个 `PASS`
- 最后一行为 `E2E smoke test completed successfully.`

## 7. 回归与发布门禁

- 任一核心检查失败，不执行对外发布。
- 若仅 CORS 失败，先核对 `.env.prod` 的 `APP_CORS_ALLOWED_ORIGINS`。
- 若登录失败，优先检查数据库初始化是否成功执行了 `schema.sql`。
