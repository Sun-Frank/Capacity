# CAPICS 服务器实机首发检查清单

## 1. 执行顺序

1. 填写生产参数：
   - `cp deploy/.env.prod.example deploy/.env.prod`
   - 修改 `deploy/.env.prod` 中所有 `CHANGE_ME`
2. 首发预检：
   - `bash deploy/first-release-precheck.sh deploy/.env.prod`
3. 首发一键执行（预检 + 部署 + 冒烟）：
   - `bash deploy/first-release-run.sh deploy/.env.prod`

## 2. 上线门禁（必须全部通过）

1. 预检结果必须 `Precheck passed`.
2. 部署日志必须出现 `Deploy completed.`
3. 冒烟日志必须出现 `E2E smoke test completed successfully.`
4. `systemctl status capics-backend` 为 `active (running)`。
5. `nginx -t` 返回 `test is successful`。

## 3. 首发后人工抽检（10 分钟内）

1. 浏览器访问 `http://<域名或IP>/`，页面可打开。
2. 使用管理员账号登录，确认可进入首页。
3. 打开以下页面确认无报错：
   - `/`
   - `/capacity-assessment`
   - `/capacity-realtime`
4. 执行一次登录态接口验证：
   - `GET /api/auth/current` 返回 200。

## 4. 回滚策略（首发失败时）

1. 后端回滚：
   - 替换 `${DEPLOY_BASE}/backend/capics-backend.jar` 为上一个可用版本
   - `systemctl restart capics-backend`
2. 前端回滚：
   - 回滚 `${NGINX_WEB_ROOT}` 到上一个静态包
   - `systemctl reload nginx`
3. 数据库说明：
   - 当前 `deploy.sh` 会执行 `schema.sql`（包含重建表），仅允许首发或确认可清库场景执行。
