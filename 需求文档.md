# 产能评估系统 - 需求文档

## 1. 系统概述

### 1.1 系统目标
开发一个产能评估系统，需要登录和权限管理，用于计算和评估生产线产能，支持MRP计划管理、产品主数据维护、工艺路线配置、生产线配置、产能计算和人力核算等功能。

### 1.2 技术栈
- **前端**: Vue 3 + Element Plus
- **后端**: Spring Boot 3.x (Java 17+)
- **数据库**: PostgreSQL
- **测试框架**: JUnit 5 + Mockito (后端) + Vitest (前端)

### 1.3 架构选择
- **架构类型**: 单体应用（所有模块在一个Spring Boot应用中）
- **认证方式**: 本地数据库（用户密码存在PostgreSQL）
- **API风格**: RESTful API

## 4. 模块详细设计

### 4.1 页面设计模块

**功能描述**: 设计系统整体UI界面和用户体验

**设计风格**: Minimalist Monochrome（极简黑白）

**设计哲学**: 极简主义设计将视觉元素精简到最本质的黑白两色和排版。没有accent色来掩饰瑕疵，没有渐变来柔和边缘，没有阴影来创造虚假深度。每一个设计决策都必须凭自身的价值站立。这是设计的纪律——克制成为表达的终极形式。

**视觉关键词**: 严肃、权威、永恒、编辑感、智慧感、戏剧性、精致、冷峻、自信、不妥协

**包含内容**:
- 系统首页/仪表盘设计
- 导航菜单结构
- 通用布局组件
- 响应式设计规范

**色彩系统（严格黑白）**:

| | PRIMARY KEY (item_number, line_code) | | | 联合主键 |

**编码匹配逻辑**:
- 导入产品时提供两个字段：产品代码（如 SC40DAGG04）和编码族（如 SC40DAGG**）
- 系统根据编码族查询关联的所有生产线配置
- 同一编码族可能在多条生产线生产，每条生产线的CT/OEE/人数可能不同
- 导入时为每条生产线生成一条产品记录，自动带出对应的 CT、OEE、生产人员数量

**版本更新机制**:
- 编码族数据更新后，已导入的产品主数据不受影响
- 只有新导入的产品才会使用最新编码族的数据
- 历史数据保留当时的版本快照

**测试策略**:
- 单元测试: ProductFamilyService、ProductService CRUD逻辑、编码匹配算法
- 集成测试: ProductController + Database
- 端到端测试: 编码族导入、产品导入、产品列表

**模块依赖**: 无

### 4.4 MRP计划管理模块

**功能描述**: 记录成品需求计划，作为产能计算的输入数据之一。

**与主数据模块的关系**: 完全独立，无关联。MRP记录成品需求，主数据记录半成品参数（CT/OEE/人数）。后续通过成品工艺路线模块（BOM）将成品拆解为半成品需求后，再结合主数据进行产能计算。

**数据来源**: 从MRP系统导入或手动维护的需求计划

**API接口设计**:

| 方法 | 端点 | 描述 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | /api/mrp/plans | 查询MRP计划列表 | - | `[MRPPlan]` |
| GET | /api/mrp/plans/{id} | 查询单个MRP计划 | - | `MRPPlan` |
| POST | /api/mrp/plans | 创建MRP计划 | `MRPPlan` | `MRPPlan` |
| PUT | /api/mrp/plans/{id} | 更新MRP计划 | `MRPPlan` | `MRPPlan` |
| DELETE | /api/mrp/plans/{id} | 删除MRP计划 | - | `{success}` |
| POST | /api/mrp/plans/import | 导入MRP计划 | `MultipartFile` | `{count}` |
| GET | /api/mrp/plans/latest | 获取最新版本计划 | `version` | `[MRPPlan]` |

**数据库表结构**:

| 表名 | 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|------|
| mrp_plan | id | BIGSERIAL | PRIMARY KEY | 计划ID |
| | item_number | VARCHAR(50) | NOT NULL | 成品物料号 |
| | description | VARCHAR(255) | | 产品名称 |
| | site | VARCHAR(50) | NOT NULL | 工厂代码 |
| | production_line | VARCHAR(100) | | 打包生产线名称 |
| | release_date | DATE | NOT NULL | 计划完成日期（汇总依据） |
| | due_date | DATE | | 交付日期 |

### 4.5 成品工艺路线模块

**功能描述**: 定义成品的BOM（物料清单），用于将成品需求拆解为半成品需求。为产能计算提供半成品需求数据。

**业务逻辑**:
1. 导入工艺路线表，记录每个成品由哪些组件构成
2. BOM耗量默认为1（每个成品需要1个组件）
3. 产能计算时，实时查询工艺路线，将成品需求展开为半成品需求
4. 组件的CT/OEE/人数通过"组件物料号+生产线"从主数据表查询

**数据来源**: 从Excel导入的工艺路线数据

**API接口设计**:

| 方法 | 端点 | 描述 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | /api/routings | 查询工艺路线列表 | - | `[Routing]` |
| GET | /api/routings/{id} | 查询单个工艺路线 | - | `Routing` |
| POST | /api/routings | 创建工艺路线 | `Routing` | `Routing` |
| PUT | /api/routings/{id} | 更新工艺路线 | `Routing` | `Routing` |
| DELETE | /api/routings/{id} | 删除工艺路线 | - | `{success}` |
| POST | /api/routings/import | 导入工艺路线 | `MultipartFile` | `{count}` |
| GET | /api/routings/by-product/{itemNumber} | 按成品查询BOM | `itemNumber` | `[RoutingItem]` |
| POST | /api/routings/expand | BOM展开（实时） | `{itemNumber, quantity}` | `ExpandedBOM` |

**BOM展开接口说明**:
- 输入：成品物料号 + 需求数量
- 输出：该成品所有层级组件的需求数量
- 实时查询主数据表获取组件的CT/OEE/人数

**数据库表结构**:

| 表名 | 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|------|
| routing | id | BIGSERIAL | PRIMARY KEY | 工艺路线ID |
| | product_number | VARCHAR(50) | NOT NULL | 成品物料号 |
| | description | VARCHAR(255) | | 产品名称 |
| | version | VARCHAR(20) | | 版本 |
| | created_by | VARCHAR(50) | | 创建人 |
| | created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| | updated_by | VARCHAR(50) | | 修改人 |
| | updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

| 表名 | 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|------|
| routing_item | id | BIGSERIAL | PRIMARY KEY | BOM项ID |
| | routing_id | BIGINT | FOREIGN KEY | 工艺路线ID |
| | component_number | VARCHAR(50) | NOT NULL | 组件物料号 |
| | line_code | VARCHAR(50) | NOT NULL | 生产线编码 |
| | bom_level | INTEGER | | BOM层级（仅展示） |
| | bom_quantity | DECIMAL(10,2) | DEFAULT 1 | BOM耗量（默认1） |
| | created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |

**BOM展开算法**:
```
输入: item_number, quantity (成品需求数量)
1. 查询工艺路线表，获取该成品所有BOM行
2. 对每行BOM:
   - 半成品需求 = quantity × bom_quantity (默认1)
   - 查询主数据表 (component_number + line_code) 获取 CT/OEE/人数
3. 返回展开结果:
   [
     { component: "ICG0472YC36N121V8C", line: "ICP1001N", quantity: 1000, ct: 45, oee: 85, workers: 2 },
     { component: "SBAI09AABZPC31", line: "SMT1001N", quantity: 1000, ct: 30, oee: 80, workers: 3 },
     ...
   ]
```

**测试策略**:
- 单元测试: RoutingService CRUD、BOM展开算法
- 集成测试: RoutingController + Database
- 端到端测试: 工艺路线列表、导入功能、BOM展开

**模块依赖**: 产品主数据模块（查询组件CT/OEE/人数）

### 4.7 生产线实时配置模块

**功能描述**: 存储生产线实时配置的计算结果。根据MRP周需求、工艺路线BOM、主数据CT/OEE/人数、生产线配置，计算班产量和周需求汇总。此模块作为产能计算的直接数据源。

**数据来源**: 实时组装MRP、工艺路线、主数据、生产线配置的数据

**计算逻辑**:
```
班产量 = 3600 / CT × OEE × 每班工作小时数
周需求 = Σ (成品需求 × BOM耗量)  // 通过工艺路线展开
```

**API接口设计**:

| 方法 | 端点 | 描述 | 请求体 | 响应 |
|------|------|------|--------|------|
| GET | /api/line-realtime/list | 查询实时配置列表 | `lineCode, version` | `[LineRealtime]` |
| GET | /api/line-realtime/{lineCode} | 查询单条生产线实时配置 | `lineCode, version` | `LineRealtime` |
| POST | /api/line-realtime/calculate | 执行计算并存储 | `{version}` | `{count}` |
| GET | /api/line-realtime/weekly-demand | 获取周需求汇总 | `lineCode, version` | `WeeklyDemand[]` |

**数据库表结构**:

| 表名 | 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|------|
| line_realtime | id | BIGSERIAL | PRIMARY KEY | ID |
| | line_code | VARCHAR(50) | NOT NULL | 生产线代码 |
| | item_number | VARCHAR(50) | NOT NULL | 成品物料号 |
| | component_number | VARCHAR(50) | NOT NULL | 组件物料号 |
| | description | VARCHAR(255) | | 产品名称 |
| | shift_output | DECIMAL(12,2) | | 班产量 |
| | shift_workers | INTEGER | | 班人数 |
| | ct | DECIMAL(10,2) | | 周期时间（秒） |
| | oee | DECIMAL(5,4) | | OEE效率（0-1） |
| | weekly_demand | JSONB | | 周需求汇总 `{"W01": 100, "W02": 200}` |
| | mrp_version | VARCHAR(10) | | MRP版本 |
| | calculated_at | TIMESTAMP | DEFAULT NOW() | 计算时间 |
| | PRIMARY KEY (line_code, item_number, component_number, mrp_version) | | | 联合主键 |

**数据结构示例**:
| line_code | item_number | component_number | shift_output | shift_workers | ct | oee | weekly_demand |
|-----------|-------------|-----------------|--------------|---------------|-----|-----|---------------|
| ROT1001N | DP10CAFBB | RODP10CAFZPB18 | 768 | 2 | 45.00 | 0.80 | `{"3/16": 200, "3/23": 0}` |
| ROT1001N | DP10CAFBB | RODP10CAFZPA18 | 768 | 2 | 45.00 | 0.80 | `{"3/16": 0, "3/23": 370}` |

**模块依赖**:

- 工艺路线模块（BOM展开）
- 主数据模块（CT/OEE/班人数）
- 生产线配置模块（每班工作时长）
