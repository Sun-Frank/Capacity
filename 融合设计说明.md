# CAPICS 融合版（V3）改造说明

## 文档信息
- 更新日期：2026-04-25
- 文档版本：v1.2.2

## 目标
基于现有 `capics_V1`（Spring Boot + Vue）和 `CAPACITY_V2`（报表模板+MySQL业务表）重构为一个统一可维护的产能评估系统。

## 本次完成的融合点
1. 补齐了手册中“静态月产能评估”能力：新增前端页面 `/capacity-assessment-monthly`。
2. 引入 V2 的“固定人力因子”思想：新增 `manpower_plan` 表并接入周/月 LOAD 计算分母。
3. 引入 V2 的“线体分类/归属”能力：新增 `line_profile` 表支持产线分类。
4. 引入 V2 的“会议纪要”能力：新增 `meeting_minutes` 表与接口。
5. 新增“融合工作台”页面 `/fusion-workbench` 管理上述融合数据。

## 核心计算变化
原公式（周）:
`LOAD = demand * CT / (工作天 * 班次 * 每班小时 * OEE * 3600)`

融合后（周/月）:
`LOAD = demand * CT / (原分母 * manpowerFactor)`

其中 `manpowerFactor` 来自 `manpower_plan`（按 lineClass + 日期向前取最近一条），默认值 `1.0`。

## 新增接口
- `GET/PUT /api/fusion/line-profiles`
- `GET/POST/DELETE /api/fusion/manpower-plans`
- `GET/POST/DELETE /api/fusion/meeting-minutes`

## 数据库变更
`schema.sql` 新增:
- `line_profile`
- `manpower_plan`
- `meeting_minutes`

并新增相关索引与 `line_profile` 初始化数据。

## 部署提示
- 后端重建：`mvn clean package -DskipTests`
- 前端重建：`npm run build`
- 首次部署需执行更新后的 `schema.sql`
