# MRP 对比页 Align

## 文档信息
- 更新日期：2026-04-25
- 文档版本：v1.2.2

## 问题定义
新增一个 `MRP对比` 页面。用户选择两个不同的 MRP 文件，按相同时间段对数量进行对比，并按 `产品描述` 维度汇总后展示差异，使用颜色区分增减变化。

## 已确认范围
- 页面新增在前端系统中，风格应与现有页面一致。
- 文件选择复用现有 MRP 筛选链路：`createdBy -> fileName -> version`。
- 对比对象是两个不同的 MRP 文件版本组合。
- 聚合维度是 `产品描述`。
- `产品描述` 字段来源于 `产品列表` 主数据，不以 MRP 文件内原始描述为准。
- 结果需要按相同时间段比较数量差异，并通过颜色区分。

## 项目上下文
- 现有前端已有 MRP 页面与筛选逻辑，可复用 `@/api/mrp` 与 `useMrpFilters` 的筛选模式。
- 现有 MRP API 已支持：
  - 查询导入人、文件名、版本
  - 获取单个文件版本的周需求明细 `getWeeklyDemandSingle`
  - 获取单个文件版本的月需求明细 `getMonthlyDemandSingle`
- 产品主数据页面已有 `getProducts` 接口，可获取 `itemNumber / description` 映射。
- 当前仓库已有新页面文档工作流，适合在 `docs/mrp-compare/` 下保留 6A 产物。

## 数据口径假设
- 对比以 `itemNumber` 先关联产品主数据中的 `description`。
- 再将同一 `description` 下的多个物料数量汇总。
- 同时间段对比逻辑：取两个文件共同拥有的时间列交集，只比较交集部分。
- 差异值定义：`文件B数量 - 文件A数量`。
- 颜色方向提议：
  - 正差异：暖色
  - 负差异：冷色
  - 零差异：中性色

## 当前建议范围
- 第一版优先支持周/月两种时间视图切换，而不是混在同一张表里。
- 第一版先做页面查询、汇总、差异展示，不做导出与持久化。
- 第一版不改后端接口，全部通过前端组合现有接口完成。

## 暂不纳入范围
- 新增后端聚合接口
- 自定义差异公式
- 导出 Excel/CSV
- 历史对比记录留存
- 多文件同时对比

## 关键待确认点
1. 新页面是否需要同时支持 `周对比` 和 `月对比`。
2. 差异颜色是否采用：`增加=红/橙，减少=蓝，持平=灰`。
3. 页面默认排序是否按 `差异绝对值` 从高到低。
