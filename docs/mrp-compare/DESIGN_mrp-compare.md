# MRP 对比页 Architect

## 文档信息
- 更新日期：2026-04-26
- 文档版本：v1.3.0

## 页面定位
这是一个分析页，不是明细维护页。目标是在两个 MRP 文件之间快速识别同一时间段、同一产品描述下的需求数量差异。

## 视觉方向
遵循现有系统页面骨架，但比普通表格页更强调差异识别：
- 延续 `page-header + filter card + content card` 结构
- 中央区域突出“对比对象”和“差异强弱”
- 配色克制，不做复杂装饰，核心依靠表格底色和差异徽标传达变化

## 建议信息架构
1. 页面标题区
- 标题：`MRP对比`
- 副标题：`按产品描述汇总两个MRP文件在相同时间段的数量差异`

2. 对比筛选区
- 左侧文件 A：导入人 / 文件 / 版本
- 右侧文件 B：导入人 / 文件 / 版本
- 视图切换：`周对比 / 月对比`
- 操作：`开始对比`

3. 汇总指标区
- 对比时间段数
- 涉及产品描述数
- 数量增加描述数
- 数量减少描述数

4. 差异主表区
- 列结构建议：
  - 产品描述
  - 时间段
  - 文件A数量
  - 文件B数量
  - 差异值
  - 差异方向
- 支持按时间段分组显示
- 行底色基于差异方向和幅度变化

5. 差异强度辅助区
- 顶部或右上角放颜色图例
- 说明颜色语义和差异计算方式

6. AI 分析区
- 触发按钮：`AI分析差异`
- 返回内容分为两栏：
  - 结论栏：AI 综合分析文本
  - 证据栏：互联网信号 + 系统内风险上下文

## 交互流
1. 用户选择文件 A 和文件 B。
2. 切换周/月对比维度。
3. 点击 `开始对比`。
4. 前端并发请求：
- 文件 A 对应单文件需求数据
- 文件 B 对应单文件需求数据
- 产品主数据列表
5. 前端用 `itemNumber` 关联产品描述并做二次聚合。
6. 提取两个文件的共同时间段，生成对比结果。
7. 渲染汇总卡片和差异表格。

## 数据流设计
### 输入
- 文件 A：`createdByA / fileNameA / versionA`
- 文件 B：`createdByB / fileNameB / versionB`
- 视图维度：`week | month`

### 数据源
- `getWeeklyDemandSingle(token, createdBy, fileName, version)`
- `getMonthlyDemandSingle(token, createdBy, fileName, version)`
- `getProducts(token)`
- `analyzeMrpCompare(token, payload)`

### AI 综合分析数据流
1. 前端构建 `mrpPayload`（汇总、时间窗口、Top差异项）。
2. 后端在 AI 调用前补充两类上下文：
- 互联网信号：市场行情、汽车电子供应链风险（运行时抓取）
- 系统内信号：变化率、集中度、冲击度、风险分值
3. AI 返回结论后，后端同时回传：
- `analysis`（分析文本）
- `externalSignals`（互联网信号列表）
- `internalRisk`（系统内风险上下文）
4. 前端以“结论 + 证据”双栏渲染。

### 前端聚合步骤
1. 拉取文件 A、B 的原始需求数据。
2. 建立 `itemNumber -> description` 映射。
3. 将缺失描述的数据归入 `未维护描述`。
4. 对每个时间段按 `description` 汇总数量。
5. 取两个文件时间段交集。
6. 生成对比行：
- `description`
- `period`
- `qtyA`
- `qtyB`
- `delta = qtyB - qtyA`
- `direction = increase | decrease | equal`
7. 计算统计卡片数据。

## 风险点
- 如果单文件数据返回结构与周/月报表不同，需要先做适配层。
- 产品主数据可能存在 `itemNumber` 缺失描述，需要显式兜底。
- 当共同时间段很少时，页面应给出提示而不是空白。

## 推荐默认策略
- 默认进入 `周对比`
- 默认按 `|差异值|` 从高到低排序
- 默认高亮前 20 条差异最明显记录
