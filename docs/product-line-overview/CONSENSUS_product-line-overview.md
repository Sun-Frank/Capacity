# CONSENSUS_product-line-overview

## 最终共识

### 页面目标

将 `/product-line` 重构为“产线一览”分析页，以工艺段为筛选主轴，展示工艺段下所有产线的产能利用情况。

### 数据口径

- 使用现有静态产能核算接口。
- 周数据来自 `GET /api/capacity-assessment`。
- 月数据来自 `GET /api/capacity-assessment/monthly`。
- 产线名称来自 `GET /api/lines`。
- 工艺段通过产线代码前三位提取。

### 页面结构共识

1. 顶部保留 MRP 三级筛选：导入人、文件、版本。
2. 提供工艺段筛选，默认支持 `全部`。
3. 周、月分开展示。
4. 周、月建议使用页签切换，避免单页过长。
5. 每个视图保留：
   - 总览指标
   - 利用率趋势图
   - 产线利用率排名图
   - 热力矩阵

### 明确不做

- 不做动态模拟口径。
- 不保留异常提示区。
- 不保留汇总明细表区。
- 不新增后端接口。

### 技术共识

1. 前端直接聚合现有接口数据，不增加后端改造。
2. 正式实现允许新增图表库。
3. 产线一览页应优先复用现有筛选和数据请求模式。

### 验收方向

页面需要让用户能够快速回答：

1. 某工艺段整体利用率高不高。
2. 某工艺段下哪些产线负荷最高。
3. 某工艺段在哪些周或月份负荷波动明显。
